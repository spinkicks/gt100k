# Loop decisions — what was chosen and why (do not re-litigate)

- 2026-07-20 (P0): Land T001–T005 as one cohesive skeleton increment because T001 introduces the aggregate gate and its Buf, Go, OPA, and Terraform lanes cannot all be green until the four smoke inputs exist. Keep the contract lane at `buf lint` and the Terraform lane on `_smoke`; T020 and T045 add breaking/freshness and per-module validation at their ordered tasks.
- 2026-07-20 (P0): Keep the task-required `proto/buf.yaml` and add a root `buf.yaml` that declares `proto` as its module. The spec's `buf lint proto` resolves the nested registry configuration, while the unattended harness invokes `buf lint` from the repository root; the root workspace file makes both supported entry points use the same module boundary without moving generated-code configuration out of `proto/`.
- 2026-07-20 (P0): Implement T006 with the official setup actions named by the spec. The mandatory job checks out full history for the future `buf breaking` comparison and installs every pinned gate tool; the separate integration job installs only pinned Go and verifies Docker because `gate-integration` is exclusively the build-tagged Go test lane. Include root `buf.yaml` and the workflow itself in the path filters because both can change CI behavior.
- 2026-07-20 (P0): Keep the task-required `proto/buf.gen.yaml` and add an identical root `buf.gen.yaml` because buf 1.50.0 resolves the generation template from the current directory, not from the `proto` input. This makes the spec's exact root command `buf generate proto` work without moving the required nested template. Pin the generated code's direct `google.golang.org/protobuf` runtime to v1.36.5, matching the pinned `protoc-gen-go` version, rather than allowing an unspecified newer runtime.
- 2026-07-20 (P1): Preserve the data model's pinned `ActorClass` wire names (`HUMAN` through `SYSTEM`) rather than renaming them to Buf's preferred `ACTOR_CLASS_*` spellings. Scope an `ENUM_VALUE_PREFIX` exception only to `envelope.proto` in both Buf entry-point configs (workspace-relative at root, module-relative under `proto/`), leaving every other STANDARD rule active. Keep P1 contracts fields-only without Protovalidate annotations because the ordered spec places Go invariants in P2 and pins no Protovalidate dependency/version.
- 2026-07-20 (P1): Preserve the data model's pinned unprefixed `AssentResponse` and `AppealStatus` wire values, extending the existing path-scoped `ENUM_VALUE_PREFIX` exception only to `assent.proto` and `appeal.proto`. Use proto3 `bool` for `AuditEntry.policy_allow`, the simplest data-model-permitted default, rather than adding an unpinned wrapper dependency; Go invariants remain deferred to their ordered tasks.
- 2026-07-20 (P1): Give each shared invariant error a deterministic sentence that includes its required diagnostic field, and implement `Error()` on a value receiver so both values and pointers satisfy `error`. Keep `Clock` and `IDGenerator` as one-method interfaces only; concrete clocks and generators belong at composition boundaries, not in the pure core.
- 2026-07-20 (P1): Make `WithEnvelope` a generic protobuf-reflection helper that deep-clones `ValidEnvelope` into any generated contract's `header` field. This keeps later contract fixtures terse without aliasing the mutable canonical protobuf. For otherwise unspecified complete-envelope values, use deterministic synthetic opaque ids, `consent_grant/1`, and synthetic model/evidence references so every one of the 12 header fields is populated.
- 2026-07-20 (P1): Store each G-AUTH JSON input beneath a unique top-level key matching its filename so all six documents can be loaded together by `opa test policies/` without base-document collisions. Represent the expired and withdrawn rows with empty `consents` arrays at the OPA boundary because the pinned data-model decision makes the Go edge pre-filter inactive grants before policy evaluation; the distinct files preserve the six-row acceptance table.
- 2026-07-20 (P2): Treat `recorded_at` as required and validate both envelope timestamps with Protobuf `CheckValid`, following the normative data-model text even though the compact G-ENV mutation table omits a `recorded_at = nil` row. Validate fields in schema-tag order; a nil envelope therefore deterministically reports `contract_id`. Complete T017 alongside T013 because the overnight harness may only commit a green gate.
- 2026-07-20 (P2): Validate a `LearnerEvent` by composing the common envelope validator, then requiring distinct event/ingest timestamps and the four fields pinned by the public contract (`event_type`, `learner_ref`, `source`, `payload_schema`). Keep `context` and event-level `evidence_refs` optional. Report equal timestamps as `*NamedFieldError{Field:"recorded_at"}` because the ingest timestamp is the second conflicting value and the spec does not pin that diagnostic.
- 2026-07-20 (P2): Validate any present `DecisionRecord.authorized_human` regardless of the `consequential` flag so no model/system actor can enter that field through a non-consequential path. Check the class before ref completeness so every `MODEL`/`SYSTEM` value returns the pinned `*AuthorityForgeryError`, even if unnamed; otherwise treat a missing ref or unspecified/unknown class as `*NamedFieldError{Field:"authorized_human"}` and allow only `HUMAN`, `GUARDIAN`, `CHILD`, and `STAFF`. `consequential` independently controls whether authority and the decision-local policy result are required.
- 2026-07-20 (P2): Make the `main` compatibility check bootstrap-aware only for Buf's exact no-schema error because this first registry branch has no `.proto` baseline on `main`; all other baseline errors propagate, and fixture tests prove that the wrapper passes additive changes but rejects removals once a baseline exists. Run generated-code comparison in an isolated temporary workspace before the spec-required harness `buf generate proto` + `git diff --exit-code`, so stale output fails without rewriting the working tree while CI retains the exact final check.
- 2026-07-20 (P3): Treat `guardian_authority=false` as an incomplete `ConsentGrant` because this child-consent contract has no alternate self-authority field; report `*NamedFieldError{Field:"guardian_authority"}`. Make `IsConsentActive` reject any malformed grant before applying the pinned inclusive-effective/exclusive-expiry window, so invalid consent can never authorize processing. Preserve the specified nil-expiry open window and treat a nil withdrawal state as not withdrawn.
- 2026-07-20 (P3): Validate `AssentRecord` fields in wire-tag order, accept only the three defined nonzero responses, validate optional `renewal_at` when present, and preserve `honorable=false` as a meaningful valid value rather than treating it as missing. Implement `AssentBlocks` as the pinned `honorable && response != ASSENT` predicate; malformed responses are rejected by `ValidateAssentRecord` at the contract boundary.
- 2026-07-20 (P3): Supersede the earlier assumption that `opa test policies/` loads `policies/testdata/`: OPA 1.4.0 omits that specially named nested directory unless it is passed explicitly. Keep the six JSON documents there for the T024 Go SDK edge, but make `authz_test.rego` self-contained with the same synthetic rows so the spec's exact gate command remains sufficient. Use the stable set member `"authority_forgery"` for the otherwise unspecified `deny_authority_forgery` reason.
- 2026-07-20 (P3): Build the authorization-only OPA artifact with pinned OPA 1.4.0 and revision `opa-bundle/2026-07-20a`, retain it beside `services/identity-consent/authz.go`, and embed it so the default Go lane evaluates a compiled bundle without filesystem or OPA-server dependencies. Treat the bundle manifest revision as authoritative and reject a decision whose `policy_version` differs from it; a second deterministic `opa build` must compare byte-for-byte.
- 2026-07-20 (P3): Accept `google.golang.org/protobuf` v1.36.6 as the Go runtime selected by the pinned OPA v1.4.0 module, which requires that patch release. The schema generator remains pinned to `protoc-gen-go` v1.36.5 as specified; forcing the older runtime with a replacement would violate Go module compatibility for the required OPA SDK.
- 2026-07-20 (P3): Make the service-local consent repository's withdrawal operation an atomic transition that returns the stored withdrawn snapshot plus a `changed` bit. The golden `WithdrawConsent(ctx, deps, contractID, at)` signature otherwise has no safe way to recover the pseudonymous subject or distinguish a new withdrawal from an idempotent repeat; T034 should preserve this compare-and-set semantic when the shared ports are centralized.
- 2026-07-21 (P3): Keep `IdentityRepository` and `EnrollmentHandoffSource` service-local until T034 centralizes the structurally identical spine ports, avoiding a premature P4 package dependency. Validate every reference-only handoff field, accept only the pinned Track A/B enum values, and validate the returned `ActorRef` ref/class/role so unknown Protobuf enum values fail closed and downstream callers never receive the enrollment record itself.
- 2026-07-21 (P3): Keep `AssentRepository` service-local through T030 with the data model's exact `Put`/`ForChild` shape until T034 centralizes the spine ports. `RecordAssent` validates then persists the caller's response unchanged, so the already-pinned `AssentBlocks` rule remains the single veto predicate; the in-memory fake deep-clones records and rejects duplicate contract ids to preserve append-only semantics.
- 2026-07-21 (P4): Use one in-memory `Store` as the T031 atomic boundary for decision, outbox, and audit state. `Commit` validates the complete unit, checks all existing and intra-unit identifiers under one lock, and only then deep-clones every record into state, making partial publication impossible without rollback machinery. Keep `Deliver` independent of event time: validate the event, deduplicate solely on `Envelope.contract_id`, apply the projection, and mark the offset only after a successful application.
- 2026-07-21 (P4): Because the spec pins `HandleCommand` behavior but not the `Command` shape, accept prepared `DecisionRecord` and `LearnerEvent` candidates plus session/purpose/subject/jurisdiction, then deep-clone them and overlay the resolved pseudonymous actor and actual policy version. Use the injected clock for authorization/staging, the event `contract_id` as the stable outbox idempotency key, and the injected ID generator for audit ids. Authorized audit stays inside the atomic `UnitOfWork`; denial performs exactly one standalone audit append. Keep identity and active-consent ports as narrow read-only command-path interfaces until T034 centralizes the full repository mutation surfaces.
- 2026-07-21 (P4): Back the T033 PostgreSQL adapter with three append-only tables (`spine_decisions`, `spine_outbox`, `spine_audit`), a partial pending-row index, Protobuf payload bytes, and validation/serialization before the transaction so the atomic decision/outbox/audit transaction stays short. Expose separate `Store` and `DecisionStore` views over one `pgxpool` because Go cannot implement the data model's two differently typed `Append` methods on one concrete type; the shared database remains the single atomic boundary.
- 2026-07-21 (P4): Use the testcontainers-go v0.35.0 reference images `postgres:16-alpine` and `docker.redpanda.com/redpandadata/redpanda:v23.3.3` for the opt-in integration proof, with the module's condition-based PostgreSQL readiness strategy. Publish `Envelope.contract_id` as the Kafka record key, exercise PostgreSQL outbox persistence and Redpanda transport for every event, and reuse the already-proven in-memory offsets/projection at the consumer edge because T036 scopes PostgreSQL to outbox/decision persistence; full shared EventBus port centralization remains ordered T034 work.
- 2026-07-21 (P4): Make `pkg/spine` the single owner of cross-service port and policy types, with identity-consent type aliases preserving its existing API and no reverse dependency. Retain narrow read-only interfaces for `HandleCommand`, preserve the previously chosen compare-and-set consent withdrawal result, and split the event transport into publish-only `EventBus` plus pull-based `EventSource`: franz-go establishes subscriptions when constructing a consumer, so a callback-style `Subscribe` method would couple the shared port to a fake-only transport model and prevent the real Redpanda producer from satisfying `EventBus`.
- 2026-07-21 (P4): Make `Relay` deterministic and fail-fast over the pending-row order: count a publish before marking, mark only after publish succeeds, and leave a publish- or mark-failed row pending so retry intentionally permits duplicate transport delivery. Model that at-least-once seam with a FIFO in-memory bus that preserves duplicate publishes, expose a separate decision-repository view over the shared memory store because Go cannot overload audit/decision `Append`, and centralize audit construction/validation while allowing an empty policy reason for non-authorization lifecycle facts.
- 2026-07-21 (P4): Translate PostgreSQL unique violations at the standalone `DecisionRepository.Append` and `AuditLog.Append` boundaries into the shared `*AppendOnlyError`, matching the in-memory adapters while hiding pgx-specific errors from callers. Use the audit `entry_id` as the error's immutable identifier, consistent with the memory audit adapter; keep outbox idempotency conflicts as persistence errors because an outbox key is not necessarily a contract id.
- 2026-07-21 (P5): Treat `OverrideRecord.evidence_refs` as required because the canonical P5 user story requires evidence and G-OVR supplies a reference, even though the compact data-model invariant list omits it. Report an empty list as `*NamedFieldError{Field:"evidence_refs"}` so the contract fails closed under SC-002.
- 2026-07-21 (P5): Apply the existing human-authority allowlist (`HUMAN`, `GUARDIAN`, `CHILD`, `STAFF`) to override approvers, count distinct approvals by pseudonymous `ActorRef.ref`, and give `MODEL`/`SYSTEM` authority-forgery precedence. For DP-3 non-listed classes, enforce the specified exactly-one named human rule and use `*NamedFieldError{Field:"approvers"}` for otherwise-unpinned cardinality failures.
- 2026-07-21 (P5): Follow the normative Appeal invariant list exactly: require the complete envelope, `appellant_role`, `target_decision`, `grounds`, `requested_remedy`, a defined nonzero status, and a named `independent_reviewer`; keep `submitted_evidence_refs`, `deadlines`, and `resolution` optional because the data model does not mark them required and the golden filed record has an empty resolution. Fail closed on unknown enum numbers, and compare reviewer independence by the pinned pseudonymous `ActorRef.ref` without inventing an additional class rule.
- 2026-07-21 (P5): Use stable OPA deny-set members `"authority_forgery"`, `"four_eyes_required"`, and `"reviewer_conflict"` for the otherwise-unpinned override/appeal reasons. Count only nonempty distinct approver refs for the four-eyes rule, apply it only to the four DP-3 governed classes, and keep model/system forgery denial independent so it also protects non-governed overrides. Require a nonempty appeal reviewer ref before comparing it to `authorized_human_ref`; structural completeness remains the Go validator's responsibility.
- 2026-07-21 (P5): Record override and appeal lifecycle facts through separate, narrow dependency structs over the shared decision, lifecycle-record, audit, and injected-id ports. Derive the audit actor and policy version from the validated contract envelope, use the new outcome or lowercase appeal status as the audit outcome, and allow an empty policy reason for these non-authorization lifecycle facts. Because the pinned ports do not define a cross-repository lifecycle transaction, validate and build the audit first, append the immutable lifecycle record, then append its audit; any adapter failure is returned to the caller rather than hidden.
- 2026-07-21 (P6): Execute deletion activities in the pinned store-erasure → crypto-shred → deletion-audit order with a three-attempt Temporal retry policy. Derive each activity's idempotency key from the stable Temporal workflow id plus operation, so replay/restart is a no-op while a genuinely new workflow can process newly created subject data. Keep persistence idempotency behind an injected backend contract, treat the crypto-shred operation as the KMS-stub seam, and report a successful retry as `Compensated` without reading wall time or generating ids in workflow code.
- 2026-07-21 (P6): Make the synthetic `EnrollmentStub` stateless and repeatable, yielding a deep clone of the canonical DP-1 fixture on every `Next` call. This keeps the default adapter deterministic and mutation-safe while the shared `EnrollmentHandoffSource` remains the cutover seam for the eventual admissions-owned source.
- 2026-07-21 (P6): Have the Terraform validation gate discover every direct child of `infra/terraform/modules/` instead of hardcoding module names, and fail if the directory contains no modules. This makes each T048 module join the gate automatically; each discovered module runs noninteractive, backend-disabled init, validate, and recursive format-check, with no plan/apply path. Override `TF_DATA_DIR` with a gate-owned temporary directory so future provider/child-module initialization cannot leave `.terraform` caches for the harness to stage.
- 2026-07-21 (P6): Adapt the shared `DeletionStarter` to Temporal through only `ExecuteWorkflow`, with the task queue injected at composition time. Use `gt100k-deletion/<subject_ref>` as the stable workflow id, `USE_EXISTING` for a running execution, and `REJECT_DUPLICATE` for a completed execution, so the consent repository's single successful withdrawal transition maps to at most one durable workflow without waiting for completion. Propagate genuine client start failures and append the withdrawal audit only after Temporal accepts the start request.
- 2026-07-21 (P6): Model the three reserved Public/Sandbox/Sensitive boundaries as empty AWS Organization OUs while creating accounts only for Core and Identity, the simplest interpretation that preserves the required isolation graph without provisioning future workloads. Keep every AWS module provider-injected and reject non-commercial-US regions; keep `event-runtime` resource-free because managed Redpanda and Temporal operations are explicitly deferred.
- 2026-07-21 (P6): Use a three-AZ private-only VPC/EKS baseline with no NAT or public Kubernetes endpoint, explicit VPC endpoint/DNS egress, explicit cluster administrators, and IRSA as the workload-identity seam. Use AWS-managed RDS master credentials, KMS encryption, Multi-AZ PostgreSQL, and 35-day PITR; expose pgvector as a required migration contract because extension installation belongs to the database migration path, not infrastructure provisioning.
- 2026-07-21 (P6): Compose the validate-only dev root with credential-free `organization`, `core`, and `identity` AWS provider aliases so account-boundary intent is explicit without real account identifiers or assume-role credentials. Instantiate IAM once for EKS base roles and again after EKS for optional IRSA roles, avoiding an OIDC dependency cycle; keep the admin ARN, OIDC thumbprint, and all managed-runtime endpoints synthetic or empty.
- 2026-07-21 (P6): Extend the Terraform gate to mirror the complete `infra/terraform` tree before validating both leaf modules and environment roots, preserving relative child-module sources while keeping one temporary `TF_DATA_DIR` per configuration. Commit the T049-required `terraform.tfvars.example` now; T055 should reconcile it as already present while making only the remaining final shared/root placeholder and ignore changes.
- 2026-07-21 (P7): Keep the T050 API guides beside the packages they document and protect their required contract names, public entry points, and deferred-work statements with package-local Go tests. State the current transaction boundaries explicitly: command decision/outbox/audit commits are atomic, while projection/offset writes and lifecycle-record/audit appends remain separate injected seams that production adapters must make idempotent or transactional.
- 2026-07-21 (P7): Compose the T051 demo from the existing enrollment, consent/assent, embedded OPA, command/outbox, consumer, lifecycle, and deletion APIs, with only demo-local in-memory repositories and deterministic clock/id adapters. Execute the real deletion workflow synchronously in Temporal's SDK test environment so `go run ./cmd/demo` proves terminal completion without a Temporal server, inject a silent logger so process output is deterministic, and require the demo backend to retain the deletion-audit fact before reporting success. Keep Terraform execution in the validate-only gate rather than spawning infrastructure tooling from the demo; the executable creates no cloud resources.
- 2026-07-21 (P7): Prove FR-016 record replay through two independent `HandleCommand` executions over fresh stores with identical injected clocks and id generators, comparing deterministic Protobuf bytes plus outbox timing/key facts. Prove workflow determinism with Temporal's real `WorkflowReplayer` over a synthetic complete 35-event deletion history, and retain an activity-order negative control so the fixture cannot pass without exercising Temporal's nondeterminism detector. Keep both proofs hermetic in the default Go lane; no Temporal server or production helper API is needed.
- 2026-07-21 (P7): Consolidate G-ENV, G-DEC, G-ASSENT, G-OVR, and G-APL inside the platform package so the golden suite can assert exact typed invariant diagnostics, while exercising G-IDEM through the public spine API and real in-memory adapters. Pin the override/appeal payloads and consequential decision authority/policy explicitly against the canonical synthetic fixtures. Retain G-AUTH in its existing self-contained Rego decision table rather than duplicating OPA semantics in Go.
- 2026-07-21 (P7): Treat T054's local operations proof as the exact non-Git `make gate` constituents plus the fresh Docker integration lane because GNU Make is unavailable and the loop reserves all Git operations for the harness. Keep `make gate` as the runbook's primary operator/CI path, document the harness-owned `main` compatibility and generated-tree diff checks explicitly, and retain the hermetic fixture/freshness checks locally rather than weakening the contract gate.
- 2026-07-21 (P7): Keep the T049 `terraform.tfvars.example` unchanged because its four synthetic assignments already match the pinned T055 contract. Add the exact `.env.local`, `**/terraform.tfvars`, and `**/*.tfstate*` rules even though `.env.*` already covers the local placeholder, so the final shared-file contract is explicit. Create `.loop-done` after the fresh non-Git and Docker gates pass; the harness remains responsible for the prohibited Git-backed `main` comparison and generated-tree diff.
