# PassionLab

The unified passion product: where a GT100K student discovers, develops, documents, and sustains a
genuine passion over years. Code lives under `passion/`. This file is the glossary — vocabulary only,
no implementation details. Terms are added/sharpened as the `grill-with-docs` session resolves them.

## Language

**Voluntary Return**:
Unprompted re-engagement with a domain *after* novelty, praise, deadlines, rewards, reminders, and
nudges have faded (read at 7 and 30 days). The trustworthy interest signal.
_Avoid_: dwell time, time-in-app, engagement, session length

**InterestHypothesis**:
The mutable, versioned evidence record that is the discovery output — a domain × work-mode read with
supporting and disconfirming evidence, never a scalar. Lifecycle: `EXPLORING → EMERGING →
CANDIDATE → ACTIVE / CONTESTED / PARKED / REOPENED`.
_Avoid_: passion score, interest rating, profile

**Revisable Hypothesis**:
The child-facing framing of an `InterestHypothesis` — always provisional and disputable, never a fixed
label ("you are a musician").
_Avoid_: verdict, label, result

**Signal Firewall**:
The rule that ambient delight (the cat, fireflies, the stove — any "juice") emits zero interest signal;
only genuine domain engagement and Voluntary Return count.

**Specialization Planner**:
The engine that, once an interest is validated, allocates a student's *afternoon* block across a
primary spine, adjacent work, and a wildcard. Revisable; never touches the morning academic block.
_Avoid_: career plan, major, track

**EvidenceGraph**:
The content-addressed (Merkle / RFC-6962) provenance DAG of a student's real project process — every
artifact, attempt, tool, AI-assist, and review — proving authentic, human-owned work.
_Avoid_: portfolio, plagiarism check, AI detector

**TimeBack**:
The external inherited mastery engine (Alpha / GT School): 1 focused minute = 1 XP, with a daily
per-section XP floor. PassionLab consumes its signals; it does not own it.
